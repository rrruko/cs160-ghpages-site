Dreaming in Code Answers

Chapter 0

Who wrote "software is hard?" Who is that guy?

Donald Knuth, author of The Art of Computer Programming and TeX.

Programmers start counting at what number?

0, sometimes.

In most programming languages, listy containers are indexed starting from 0. This tradition derives from the implementation details of array indexing and related mathematical phenomena; in other words, it often makes more sense to start from 0 in computer science.

For instance: we access elements of an array a by writing a[n], which desugars to *(a + n). But because *a just refers to the start of the array, i.e., the first element, and because *(a + 0) = *a, thus a[0] accesses that element.

For instance, given a = ['A', 'B', 'C'], we have a[0] = 'A'. The first element is the one corresponding to the label 0.

It's tempting to say that programmers should just stop being weird and count from 1 like everyone else. Languages like Lua enforce this perspective by numbering arrays from 1. But that isn't a perfect solution, because counting is hard.

How many fenceposts do you need to make a 10-foot fence with posts 1 foot apart? If you don't think about it carefully, you might answer 10. We think of it as an easy problem, but even counting can be hard sometimes.

The real world is governed by math, and math is hard. When we really engage with math we reveal the perilous complexities inherent in the real world. As humans, we construct abstractions to help avoid these complexities, and as programmers, it is our job to translate between these complexities and the human intution. The translation is usually lossy.

What was the original sense of a "hacker?"

"Hacker" originally had nothing to do with computer security. A hacker, in the original sense, was someone who embodied the virtues of computer culture, and was associated with an attitude of creativity and playfulness with respect to software.

According to a 2002 NIST study what % of software came in significantly late, over budget, or was canceled?

Two out of three.

Who wrote the 1987 essay entitled “No Silver Bullet?”

Fred Brooks.

"No silver bullet" means that there is essential complexity inherent in programming that can't easily be solved by the use of a better tool. At the time, hardware was improving rapidly and some people expected the same rate of improvement in software. Brooks claimed that this wasn't possible because software was in some sense harder.

Chapter 1

What roles in the Chandler project did Michael Toy, John Anderson, Ted Burgess, Mitchell Kapor, and Lou Montulli hold.

Toy was manager. Anderson was systems architect and the de facto lead coder. Kapor was founder of the Open Source Applications Foundation. Montulli and Burgess were programmers.

What is “Bugzilla?”

A bug tracking program that the team used to record estimated times to fix bugs.

What is OSAF?

The Open Source Applications Foundation, the organization in charge of the Chandler project.

What is the projects name?

Chandler.

What will the software do?

It was supposed to be a "personal information manager" for organizing calendars, emails, and to-do lists.

What is Toy’s keyword for “black hole” bugs?

"Scary".

What scared Toy so much about Bug 44?

It had a "black hole-like quality", because nobody could estimate how long it would take to fix.

What did Toy refer to as a “snake?”

An important problem that the team didn't have consensus on how to attack.

In the software world, what does “slippage” mean?

Lateness.

Fredrick Brooks was a programming manager for what software project?

The IBM System/360.

What is Brooks's Law?

"Adding manpower to a late software project makes it later."

Brooks found what % of project time was spent writing code?

One-sixth.

Brooks found what % of project time was for testing and fixing bugs?

One-half.

Brooks observed that the unit of effort named “man-month” only applied under what conditions?

Only when a task can be partitioned among many workers with no communication among them. Adding new people to a project often costs so much more extra time spent communicating with the new members that it isn't worth the time saved by splitting up work.

What is the difference between source code and the program you install (.exe) on your computer?

Executable files consist of machine code, which consists of instructions that can be directly interpreted by the processor. They are almost never written directly by a human, but instead generated by running a compiler on source code written in a more expressive and human-readable language. Such as INTERCAL.

What is the one “article of faith” that all “open source” or “free” software advocates share?

That if the source code of a program is freely available, it will eventually become better than any closed software.

What is the difference between a “good” programmer and a “great” programmer?

"Good programmers know what to write. Great ones know what to rewrite (and reuse)."

Eric Raymond’s book “The Cathedral and the Bazaar” made a distinction between two important project development ideas, briefly contrast them.

Cathedral–Release it when it's finished. Keep the team small and their agendas similar.
Bazaar–Release early and often. Delegate everything to people with all kinds of different agendas and approaches.

Has “open source” software project development refuted Brooks’s “mythical man-month” concerns?

The internet allows developers to make information about their projects easily accessible to anyone. This means less time is spent bringing new developers up to speed, and it becomes genuinely beneficial to have hundreds or thousands of contributors.

What was Andy Hertzfeld’s input when the Chandler project appeared to have stalled?

Hertzfeld advocated getting draft versions of the project up and running as quickly as possible before working out all the design details.
