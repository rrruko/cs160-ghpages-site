Chapter 11

Briefly describe the four gates explained in Chapter 11. Describe the behavior of each gate based on the input values to the gate.

OR gate: output 1 if any inputs are 1, otherwise 0.
AND gate: output 1 if all inputs are 1, otherwise 0.
NOR gate: output 1 if no inputs are 1, otherwise 0.
NAND gate: output 1 unless all inputs are 1.

Chapter 12

A half adder is built from how many "sub components" and has how many inputs and outputs?

Two components: an AND gate and an XOR gate (which is itself probably composed of an OR, a NAND, and an AND gate).

Two inputs (one for each bit) and two outputs, for the sum and the carry.

A full adder is built from how many "sub components" and has how many inputs and outputs?

Three components: two half adders and an OR gate.

Three inputs and two outputs for sum and carry.

How many total inputs and outputs are there for an 8-bit adding circuit?

Seventeen inputs for each bit of the two 8-bit inputs, plus an extra because we're using full adders. Nine outputs for each bit of the output. The ninth output might be considered extraneous too since computers add with overflow, writing the result of an 8-bit addition to yet another 8-bit number, even if the result is incorrect*. However, some (most? all?) CPUs keep track of whether the last arithmetic operation resulted in a carry, making the carry out bit useful to machine code.

*The statement 255 + 1 = 0 isn't actually wrong if you're talking about addition modulo 256.

How many total transistors are needed for the ripple version of the 8-bit adder?

144.

Chapter 15

Define byte.

A byte is generally the smallest addressable unit of information in a computer system and can usually be assumed to be 8 bits in length.

What is the range of an unsigned byte? A signed two's complement byte?

An unsigned byte ranges from 0 to 255; a signed byte ranges from -128 to 127.

What does the X in the following binary number represent 1010X10101?

The X represents 32 (2^6). 1010110101 is 32 greater than 1010010101.

What is the base 16 number system called?

Hexadecimal, from the Greek hex for six and the Latin decimus for ten.

Why is base 16 a good choice for representing binary numbers?

Since 16 is a power of two, the full range of a byte can be expressed in exactly two hexadecimal digits, e.g. hexadecimal ff is binary 11111111.

What base 16 number is 101011000011? What is an easy way to determine this without using a calculator?

Each group of 4 bits corresponds to a hex digit. In 1010 1100 0011, the bits correspond to 8 + 2, 8 + 4, and 1 + 2, i.e. 10, 12, and 3. Written in hex, those values are a, c and 3. So the hex representation is ac3.

What does the X represent in the following base 16 number D4AXF0?

The X represents 256 (16^2).

The book describes how to convert decimal to base 2 or base 16. Research and describe a technique that allows conversion of decimal to any base (2-36).

I wrote code to do that here.

The technique is this: append the number modulo the base to the string of digits. Then integer-divide the number by the base. Repeat until the number is 0.

There is a special case where the number starts out as 0; when that happens, its representation in any base is just 0.

Chapter 16

What do the prefix names kilo, mega, giga, tera, and peta represent in bytes and what were the Greek origins of the words?

10001, 10002, 10003, 10004, and 10005 respectively.

Kilo- is derived from the Greek word for "thousand". Mega-, giga-, and tera- are derived from Greek words meaning "great", "giant", and "monster" respectively, and peta- has the same etymology as penta-, coming from the Greek πέντε for "five".

At the end of Chapter 16 the author reminds the reader about something that is "very important," what is this?

RAM requires a constant flow of electricity. When it loses power, all the information it contains is lost too.
