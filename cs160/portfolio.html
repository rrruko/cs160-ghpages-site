Robot Treasure Hunt

I used the python library plotly to plot my algorithms (note: it is a log plot.) The source code is here.

Algorithms used (positive numbers represent traveling a number of meters to the right, negative numbers represent going left):

  • Linear: +1, -2, +3, -4, +5, -6 ...
  • Polynomial: +1, -4, +9, -16, +25, -36 ...
  • Exponential: +1, -2, +4, -8, +16, -32 ...
  • Factorial: +1, -2, +6, -24, +120, -720 ...
I predicted that the exponential algorithm would have the best performance. It definitely has much better performance than either the linear or polynomial algorithms, but it is less clear whether it beats the factorial algorithm.
